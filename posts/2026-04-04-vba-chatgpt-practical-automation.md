## 🔍 고품질 VBA 코드를 뽑는 프롬프트 5원칙

**챗gpt 엑셀 매크로** 품질의 80%는 프롬프트가 결정합니다. 같은 작업을 요청해도 어떻게 설명하느냐에 따라 "바로 쓸 수 있는 코드"와 "다시 물어봐야 하는 코드"로 갈립니다. 5년간 VBA를 혼자 배워도 안 되던 게, 프롬프트 한 줄 바꿔서 5분 만에 해결되는 경우를 많이 봤거든요.

### 원칙 1~3: 데이터 구조, 목표, 예외를 명시하라

**① 데이터 구조를 컬럼 단위로 설명하라**

나쁜 프롬프트: "엑셀에서 데이터 정리해주는 매크로 만들어줘"

좋은 프롬프트: "Sheet1의 A열에 날짜(YYYY-MM-DD 형식), B열에 거래처명(텍스트), C열에 금액(숫자), D열에 담당자(텍스트)가 있어. 2행부터 데이터가 시작돼."

데이터가 어디서 시작하고, 각 컬럼에 무엇이 있는지 알려줘야 코드가 정확해집니다.

**② 원하는 결과물을 "Before → After"로 설명하라**

"거래처별로 금액 합계를 Sheet2에 정리해줘"보다는 "Sheet2의 A열에 거래처명(중복 없이), B열에 해당 거래처의 C열 금액 합계가 들어가면 돼"가 훨씬 정확한 코드를 줍니다.

**③ 예외 상황을 미리 말해라**

"빈 행이 있을 수 있어", "금액에 쉼표가 포함된 텍스트 형식일 수 있어", "날짜가 없는 행은 건너뛰어야 해" 같은 예외 상황을 미리 알려주면 오류 처리 코드까지 포함된 완성도 높은 VBA가 나옵니다.

### 원칙 4~5: 엑셀 버전과 출력 형식을 지정하라

**④ 엑셀 버전을 명시하라**

Microsoft 365와 Excel 2019는 일부 함수에서 차이가 있습니다. "Microsoft 365 기준으로 작성해줘" 또는 "Excel 2019 호환으로 만들어줘"를 반드시 추가하세요.

**⑤ 코드 설명을 요청하라**

"코드에 한국어 주석(')을 달아줘"를 추가하면 각 줄이 무슨 역할을 하는지 이해할 수 있어요. 나중에 수정이 필요할 때 훨씬 편합니다. 이것만으로도 VBA 학습 효과까지 덤으로 얻을 수 있죠.

**황금 프롬프트 템플릿:**

```
[엑셀 버전]: Microsoft 365

[데이터 구조]:
- Sheet1, 1행은 헤더(날짜/거래처/금액/담당자), 2행부터 데이터
- A열: 날짜(YYYY-MM-DD), B열: 거래처명, C열: 금액(숫자), D열: 담당자

[원하는 결과]:
- Sheet2에 거래처별 금액 합계를 정리
- A열: 거래처명(중복 없이), B열: 합계금액

[예외 처리]:
- 빈 행 무시
- C열이 텍스트면 숫자로 변환 후 합산

[추가 요청]:
- 코드에 한국어 주석 추가
- 실행 완료 시 "완료!" 메시지 박스 표시
```

```vba
    Dim i As Long
    Dim processedCount As Long
    
    Set ws = ThisWorkbook.Sheets("Sheet1")
    lastRow = ws.Cells(ws.Rows.Count, "A").End(xlUp).Row
    processedCount = 0
    
    ' 아래에서 위로 순회하여 행 삭제 시 인덱스 오류 방지
    For i = lastRow To 2 Step -1
        ' 빈 행 삭제
        If WorksheetFunction.CountA(ws.Rows(i)) = 0 Then
            ws.Rows(i).Delete
        Else
            ' C열 금액 정리: 쉼표, "원" 제거 후 숫자 변환
            ws.Cells(i, 3).Value = CDbl(Replace(Replace(ws.Cells(i, 3).Text, ",", ""), "원", ""))
            processedCount = processedCount + 1
        End If
    Next i
    
    ' 중복 제거 (B열 기준)
    ws.Range("A1").CurrentRegion.RemoveDuplicates Columns:=2, Header:=xlYes
    
    MsgBox "데이터 정리 완료! 총 " & processedCount & "행 처리", vbInformation
End Sub
```

   - 탭이 없다면: [파일] → [옵션] → [리본 사용자 지정] → '개발 도구' 체크

3. **좌측 프로젝트 창** → [VBAProject] 우클릭 → [삽입] → [모듈]
4. **빈 편집창에 코드 붙여넣기**
5. **F5 키로 실행** 또는 편집창 상단 [▶ 실행] 버튼 클릭

## 🔍 실전 매크로 2: 월별 보고서 자동 생성

이건 제가 직접 써보고 "이게 된다고?" 했던 매크로예요. 원본 데이터 시트 하나를 기반으로 월별 탭을 자동으로 만들고, 각 탭에 해당 월 데이터만 필터링해서 넣어주는 작업입니다. 수작업으로는 12개 탭을 만들고 각각 필터 걸고 복사·붙여넣기 해야 하니 족히 2~3시간이 걸리는데, 매크로로는 30초예요.

### 핵심 프롬프트 설계

```
Microsoft 365 Excel VBA 코드 만들어줘.

[원본 데이터]:
- "원본" 시트, A열: 날짜(YYYY-MM-DD), B~F열: 각종 데이터, 1행 헤더

[원하는 동작]:
1. 원본 시트의 날짜 기준으로 존재하는 월(月)을 자동 파악
2. 각 월별로 새 시트 생성 (이름: "2025-01", "2025-02" 형식)
3. 해당 월 데이터만 각 시트에 복사 (헤더 포함)
4. 이미 시트가 있으면 덮어쓰기
5. 모든 월 처리 후 "총 X개 월 보고서 생성 완료" 메시지

한국어 주석 포함해줘.
```

### 생성된 코드 핵심 로직 해설

ChatGPT o3가 만드는 코드의 핵심 흐름을 이해하면 나중에 수정도 쉬워져요:

```vba
' 1단계: 원본 데이터에서 고유 월 목록 추출
Dim months As Collection
Set months = New Collection

' 2단계: 각 월별 시트 생성 (중복 방지 처리 포함)
For Each m In months
    On Error Resume Next
    Set ws = ThisWorkbook.Sheets(m)
    If ws Is Nothing Then
        Set ws = ThisWorkbook.Sheets.Add
        ws.Name = m
    End If
    On Error GoTo 0
Next m

' 3단계: 날짜 필터링 후 해당 시트로 데이터 복사
```

이 구조를 이해하면, 예를 들어 "월별"이 아니라 "팀별"로 시트를 나누고 싶을 때 ChatGPT에 "위 코드를 D열(팀명) 기준으로 시트를 나누도록 수정해줘"라고 요청하면 바로 변형 버전을 받을 수 있어요.

> 💡 **실전 팁**: 보고서 시트에 피벗테이블이나 차트까지 자동으로 넣고 싶다면 "각 월 시트에 B열~C열 데이터로 막대 차트를 자동 삽입하고, 차트 제목은 해당 월명으로 설정해줘"를 프롬프트에 추가하세요. o3는 차트 VBA도 잘 만들어냅니다.

## 🔍 실전 매크로 3: 조건부 알림 & 색상 강조 자동화

세 번째는 가장 실용적이면서 가장 자주 쓰이는 유형이에요. 납기일이 3일 이내로 남은 항목을 빨간색으로 표시하거나, 재고가 특정 수량 이하일 때 담당자에게 Outlook 이메일을 자동 발송하는 것들이죠. 엑셀 조건부 서식으로는 정적 조건만 가능한데, VBA는 동적 조건과 외부 액션(이메일 발송 등)까지 연결됩니다.

### 납기일 D-3 자동 색상 강조 매크로

```
Microsoft 365 Excel VBA 코드 만들어줘.

Sheet1의 A열: 프로젝트명, B열: 납기일(YYYY-MM-DD)

[조건]:
- 오늘 기준 납기일까지 3일 이하: 해당 행 전체를 빨간색 배경(RGB 255,0,0)으로
- 7일 이하: 주황색 배경(RGB 255,165,0)으로
- 이미 지난 납기일: 회색 배경(RGB 192,192,192)에 취소선
- 조건에 해당 없으면 배경색 없음(초기화)

매크로 실행 시마다 전체 재계산.
완료 후 "X건 주의 항목 발견" 메시지 표시.
한국어 주석 포함.
```

### Outlook 이메일 자동 발송 연동 (심화)

이 매크로는 Windows 환경 + Microsoft Outlook 설치 필수입니다. Mac에서는 동작하지 않아요. 프롬프트 예시:

```
위 납기일 코드에 추가:
납기일 3일 이하 항목이 있을 때 Outlook으로 이메일 자동 발송.
- 수신자: manager@company.com
- 제목: "[긴급] 납기일 임박 항목 X건"
- 본문: 해당 프로젝트명과 납기일 목록을 표 형태로 포함
Outlook 객체를 이용해서 만들어줘.
```

ChatGPT o3는 `CreateObject("Outlook.Application")` 구문을 활용한 완성도 높은 코드를 생성해줍니다.

> 💡 **실전 팁**: 이 매크로를 "파일 열기 이벤트"에 연결하면 엑셀 파일을 열 때마다 자동으로 실행됩니다. VBA 편집기에서 "ThisWorkbook" 더블클릭 → `Workbook_Open()` 이벤트에 매크로 호출 코드를 넣으면 돼요. ChatGPT에 "이 매크로를 파일 열기 시 자동 실행되도록 연결해줘"라고 요청하면 전체 코드를 다시 작성해줍니다.

## 🔍 실제 도입 사례: 시간이 돈이 된 현장

### 사례 1: 물류 스타트업 재무팀 (서울, 직원 50명)

서울 소재 물류 스타트업 A사의 재무팀(3명)은 매월 말일 200개 이상 거래처의 정산 데이터를 수작업으로 취합하고 있었습니다. 여러 담당자가 각기 다른 형식으로 제출하는 엑셀 파일 15~20개를 하나로 합치고, 중복을 제거하고, 거래처별 합산 보고서를 만드는 데 매월 평균 **28시간**이 소요됐어요.

2025년 10월, ChatGPT o3로 데이터 취합·중복 제거·월별 보고서 생성 매크로 3개를 제작했습니다. 소요 시간: 프롬프트 작성 및 테스트 포함 **총 4시간**. 이후 월말 정산 작업 시간은 **2.5시간으로 단축**됐습니다. 기존 대비 **91% 시간 절감**이에요. 재무팀장 표현을 빌리면 "이제 월말이 무섭지 않다"고 합니다.

### 사례 2: 제조업 영업팀 (부산, 직원 120명)

부산 제조업체 B사의 영업팀은 주 단위로 전국 20개 거점의 재고 현황 엑셀을 취합해 본사 보고서를 작성했습니다. 이 과정에서 재고 임계치 이하 항목을 수동으로 확인하고 담당 지역장에게 개별 이메일을 보내는 데 매주 **6~8시간**이 걸렸어요.

ChatGPT로 조건부 색상 강조 + Outlook 자동 이메일 발송 매크로를 도입한 후, 같은 작업이 **40분**으로 줄었습니다. 연간으로 환산하면 약 **360시간 절감**이에요. 직원 연봉 기준 시급으로 계산하면 연 **1,500만원 이상의 인건비 절약** 효과라는 게 B사의 분석입니다.

## 🔍 초보자가 반드시 피해야 할 함정 4가지

### 함정 1: 시트명과 컬럼 위치를 모호하게 설명한다

"데이터가 있는 시트에서"가 아니라 "Sheet1에서" / "첫 번째 시트에서"처럼 명확하게 지정해야 해요. ChatGPT가 시트명을 임의로 가정하면, 실제 파일과 달라서 "시트를 찾을 수 없습니다" 오류가 납니다. **시트 이름과 헤더 행 위치(몇 번째 행부터 데이터인지)는 반드시 명시하세요.**

### 함정 2: 생성된 코드를 검토 없이 바로 실행한다

ChatGPT가 생성한 코드를 실행하기 전에 반드시 **원본 파일의 백업 복사본을 먼저 저장하세요**. VBA 매크로 실행은 Ctrl+Z(실행 취소)가 되지 않는 경우가 많습니다. 행 삭제, 시트 삭제 등 되돌릴 수 없는 작업이 포함된 매크로라면 특히 주의가 필요해요. 무조건 테스트 파일에서 먼저 실행하는 습관을 들이세요.

### 함정 3: 오류를 보고 포기한다

처음 실행했을 때 오류가 나는 건 정상이에요. 중요한 건 그 오류 메시지를 정확히 복사해서 ChatGPT에 다시 붙여넣는 거예요. "오류 메시지 그대로 + 어느 줄에서 났는지" + "내 엑셀 버전"을 함께 알려주면 o3가 정확히 수정해줍니다. 평균 1~2번의 피드백 루프면 대부분 해결돼요.

### 함정 4: 한 번에 너무 복잡한 매크로를 요청한다

"데이터 정리하고, 보고서 만들고, 이메일도 보내고, PDF로 저장까지 해줘"를 한 번에 요청하면 코드가 복잡해져서 오류 추적이 어려워집니다. **기능별로 별도 매크로를 만들고, 나중에 하나의 "마스터 매크로"에서 각각을 호출하는 방식**이 훨씬 안정적이에요. ChatGPT에 "각 기능을 별도 서브루틴으로 분리해줘"라고 요청하면 됩니다.

## ❓ 자주 묻는 질문

**Q1: ChatGPT로 만든 VBA 코드, 엑셀에 어떻게 넣나요?**

엑셀 상단 메뉴에서 [개발 도구] 탭 → [Visual Basic] 클릭 → 좌측 프로젝트 창에서 [모듈] 우클릭 → [모듈 삽입] → 빈 편집창에 ChatGPT가 생성한 VBA 코드를 그대로 붙여넣기 → F5 또는 [실행] 버튼을 누르면 바로 실행됩니다. 개발 도구 탭이 보이지 않으면 [파일] → [옵션] → [리본 사용자 지정] → '개발 도구' 체크박스를 활성화하면 됩니다. 처음 한 번만 설정하면 이후에는 바로 접근 가능합니다.

**Q2: ChatGPT o3가 만든 VBA 코드가 오류가 나면 어떻게 하나요?**

오류 메시지 전체를 복사해서 ChatGPT 대화창에 "위 코드를 실행했더니 다음 오류가 납니다: [오류 메시지]. 수정해주세요."라고 붙여넣으면 됩니다. ChatGPT o3는 오류 컨텍스트를 기억하고 있어 수정된 코드를 바로 내줍니다. 보통 1~2회 피드백 루프로 해결됩니다. 오류가 반복된다면 엑셀 버전(2019, Microsoft 365 등)과 운영체제(Windows/Mac)를 함께 알려주면 정확도가 높아집니다.

**Q3: 챗GPT 엑셀 자동화, 유료 플랜이어야만 되나요?**

ChatGPT o3 모델은 2026년 4월 기준 ChatGPT Plus(월 $20) 이상 플랜에서 사용 가능합니다. 무료 플랜의 GPT-4o 미니로도 기본 VBA 코드 생성은 됩니다. 다만 복잡한 조건 분기 로직, 다중 시트 연동, 대용량 데이터 처리 코드의 정확도는 o3가 훨씬 높습니다. 업무 시간 절감 효과를 고려하면 월 $20 투자는 대부분의 직장인에게 빠르게 회수됩니다.

**Q4: 엑셀 매크로 보안 경고가 뜨는데 무시해도 되나요?**

엑셀은 외부에서 가져온 매크로 파일(.xlsm)을 열 때 보안 경고를 표시합니다. ChatGPT로 직접 생성한 코드를 본인이 VBA 편집기에 직접 붙여넣는 경우라면 '콘텐츠 사용'을 눌러도 안전합니다. 단, 인터넷에서 다운받은 .xlsm 파일에 포함된 매크로는 반드시 코드를 확인한 뒤 실행하세요. VBA 코드 안에 Shell 명령어나 외부 URL 호출 구문이 있다면 주의가 필요합니다.

**Q5: ChatGPT로 만든 엑셀 매크로, 회사 보안 정책에 걸릴 수 있나요?**

회사 데이터를 ChatGPT에 직접 입력하는 것은 기업 보안 정책에 따라 제한될 수 있습니다. 이 경우 실제 데이터 대신 구조만 동일한 가상 샘플 데이터로 프롬프트를 작성하면 됩니다. "A열에 날짜, B열에 거래처명, C열에 금액이 있는 엑셀에서 거래처별 합계를 D열에 자동 입력하는 VBA 코드 만들어줘"처럼 데이터 구조만 설명하면 실제 정보 없이도 완벽한 코드를 받을 수 있습니다.
