## Make Integromat이 뭔지, n8n·자피어와 어떻게 다른가

Make(구 Integromat)는 체코 스타트업 Celonis에 인수된 이후 2022년 리브랜딩된 노코드 자동화 플랫폼입니다. 2026년 4월 기준 전 세계 220만 명 이상이 사용하고 있으며, 연결 가능한 앱(이하 '앱 연동')은 1,700개를 넘어섰습니다(출처: [Make 공식 사이트](https://www.make.com/en/integrations)).

핵심 개념은 딱 하나입니다. **"시나리오(Scenario)"**. Make에서는 자동화 흐름을 '시나리오'라고 부릅니다. 구글 폼 응답이 들어오면 → 슬랙에 알림을 보내고 → 트렐로에 카드를 만드는 흐름이 하나의 시나리오예요.

### Make vs n8n vs 자피어 — 어떤 게 나에게 맞나요?

세 도구는 목표는 같지만 성격이 완전히 다릅니다.

| 항목 | Make | n8n | 자피어(Zapier) |
|------|------|-----|----------------|
| 난이도 | ★★☆ (중하) | ★★★ (중상) | ★☆☆ (하) |
| 코드 필요 여부 | 없음 (선택적) | 자주 필요 | 없음 |
| 연동 앱 수 | 1,700개+ | 400개+ | 6,000개+ |
| 무료 플랜 오퍼레이션 | 1,000/월 | 셀프호스팅 무제한 | 100 Task/월 |
| 시각적 빌더 | 최상 (캔버스형) | 보통 (노드형) | 보통 (리스트형) |
| 국내 사용자 지원 | 영어 (도움말 한국어 부분 지원) | 영어 | 영어 |
| 추천 대상 | 비개발자, 재택근무자 | 개발자, IT팀 | 완전 초보자 |

자피어(Zapier)가 앱 수는 가장 많지만, 무료 플랜의 100 Task 제한은 현실적으로 너무 빠듯합니다. 구글 폼 하나만 연결해도 금방 소진되거든요. n8n은 강력하지만, 복잡한 로직에서는 JavaScript 코드를 쓰는 경우가 많아 비개발자에게 진입 장벽이 있어요.

**Make는 그 중간 지점**을 정확히 찾아낸 도구입니다. 시각적으로 보기 좋은 캔버스형 빌더에서 모듈을 연결하면 되고, 필요할 때만 수식이나 간단한 텍스트 파싱 기능을 쓰면 됩니다. 코드를 전혀 몰라도 웬만한 자동화는 다 됩니다.

> 💡 **실전 팁**: Make를 처음 시작할 때 "시나리오 만들기"를 누르기 전, [Make 공식 템플릿 갤러리](https://www.make.com/en/templates)부터 둘러보세요. 이미 수천 개의 검증된 시나리오가 있고, 클릭 한 번으로 내 계정에 복사해 바로 쓸 수 있습니다.

> 🔗 **Make 공식 사이트에서 가격 확인하기** → [https://www.make.com/en/pricing](https://www.make.com/en/pricing)

*오퍼레이션(Operation): 시나리오 내 모듈이 한 번 실행될 때마다 1회 차감됩니다.*

### 오퍼레이션이 뭔지 왜 중요한가

Make의 과금 단위인 '오퍼레이션'이 처음엔 헷갈릴 수 있어요. 쉽게 설명하면 이렇습니다.

구글 폼 → 구글 시트 저장 → 슬랙 알림 전송, 이 세 단계짜리 시나리오가 한 번 실행되면 오퍼레이션 3회가 소비됩니다. 하루에 이 자동화가 10번 돌아가면 30회. 한 달이면 900회. 무료 플랜(1,000회)으로 아슬아슬하게 됩니다.

하지만 구글 폼을 하루 20~30건씩 받는다면? 바로 Core 플랜($9/월)으로 넘어가는 게 낫습니다. 월 13,000원 정도인데, 이 투자로 매일 30분씩 아낄 수 있다면 계산은 명확하죠.

> 💡 **실전 팁**: 무료 플랜에서 처음 14일은 Make의 Pro 플랜 기능을 무료 체험할 수 있습니다. 계정 생성 직후 2주 동안 고급 기능을 충분히 써보고 필요한 플랜을 결정하세요.

## Make 계정 세팅 첫 5단계 — 가입부터 첫 시나리오까지

글을 읽는 것보다 직접 따라 하는 게 빠릅니다. 아래 순서대로 따라 하면 15분 안에 첫 자동화가 작동합니다.

### 계정 생성과 기본 화면 이해하기

**1단계**: [make.com](https://www.make.com)에 접속 → "Get started free" 클릭 → 구글 계정으로 소셜 로그인 (가장 빠릅니다).

**2단계**: 대시보드 진입. 화면 왼쪽에 'Scenarios', 'Connections', 'Webhooks', 'Templates' 메뉴가 있습니다. 지금 당장은 **Scenarios(시나리오)**와 **Connections(연결)**만 알면 됩니다.

**3단계**: Connections 메뉴로 이동 → 자신이 사용하는 앱 미리 연결. 구글 워크스페이스(Gmail, Google Sheets, Google Drive), 슬랙, 노션 중 쓰는 것들을 먼저 연결해 두세요. 한 번 연결해두면 모든 시나리오에서 재사용됩니다.

### 첫 시나리오를 만드는 방법

**4단계**: Scenarios → "+ Create a new scenario" 클릭. 캔버스(흰 빈 화면)가 열립니다. 화면 중앙의 큰 동그라미(+)가 트리거 모듈입니다.

**5단계**: 트리거 동그라미 클릭 → 앱 검색 → 원하는 앱 선택 → 트리거 유형 선택. 예를 들어 "Google Forms → New Response"를 선택하면 구글 폼에 새 응답이 들어올 때마다 시나리오가 실행됩니다.

트리거 모듈 오른쪽의 작은 동그라미(+)를 클릭하면 다음 모듈(액션)을 추가할 수 있습니다. 이걸 계속 연결하면 자동화 흐름이 완성됩니다.

> 💡 **실전 팁**: 시나리오를 처음 만들 때 오른쪽 상단의 "Run once" 버튼을 반드시 활용하세요. 실제 스케줄 실행 전에 시나리오가 올바르게 작동하는지 테스트해볼 수 있고, 각 모듈의 입출력 데이터를 시각적으로 확인할 수 있습니다.

## 재택근무 반복 업무 자동화 실전 — 3가지 시나리오 세팅법

이제 본론입니다. 재택근무자가 가장 자주 반복하는 3가지 업무를 Make로 자동화하는 방법을 단계별로 알려드립니다. 각각 10분 내로 세팅 가능합니다.

### 자동화 1번: 구글 폼 응답 → 슬랙 알림 + 구글 시트 기록 (동시에)

**상황**: 클라이언트가 견적 문의 폼을 제출할 때마다, 슬랙 #영업팀 채널에 알림을 보내고 구글 시트에도 자동 기록해야 한다.

**기존 방식**: 구글 폼 응답 탭을 수시로 확인 → 복사 → 시트에 붙여넣기 → 슬랙에 직접 메시지 입력. 건당 5분, 하루 10건이면 50분.

**Make 세팅 순서**:
1. 트리거: **Google Forms → Watch Responses** (새 응답 감지)
2. 액션 1: **Slack → Create a Message** (채널 선택, 메시지 내용에 폼 응답 데이터 매핑)
3. 액션 2: **Google Sheets → Add a Row** (시트 선택, 각 열에 폼 응답 필드 매핑)

**포인트**: 모듈 2개와 3개가 병렬이 아닌 직렬로 연결됩니다. 폼 응답 1건당 오퍼레이션 3회 소비. 하루 10건이면 30회, 한 달 900회로 무료 플랜 범위 내입니다.

### 자동화 2번: 매일 오전 9시 슬랙에 '오늘의 할 일' 자동 전송

**상황**: 매일 아침 구글 시트의 태스크 목록을 보고 슬랙 DM이나 특정 채널에 오늘 해야 할 일 목록을 직접 복붙하고 있다.

**Make 세팅 순서**:
1. 트리거: **Schedule → Every day** (실행 시간: 오전 9:00 KST → UTC+9 기준 설정)
2. 액션 1: **Google Sheets → Search Rows** (오늘 날짜와 일치하는 행 필터링)
3. 액션 2: **Text aggregator** (여러 행의 데이터를 하나의 텍스트 블록으로 합치기)
4. 액션 3: **Slack → Create a Message** (정리된 할 일 목록 전송)

**포인트**: Make의 'Text aggregator' 모듈이 핵심입니다. 구글 시트에서 여러 행을 가져와 하나의 슬랙 메시지로 합쳐주는 역할을 합니다. 이 모듈을 찾아 쓸 수 있으면 Make 중급자입니다.

### 자동화 3번: 이메일 첨부파일 → 구글 드라이브 자동 저장 + 노션 로그 기록

**상황**: 거래처에서 계약서, 인보이스 등 PDF가 첨부된 이메일이 오면, 매번 저장 폴더를 열고 파일을 다운로드한 뒤 노션 DB에 수기로 기록해야 한다.

**Make 세팅 순서**:
1. 트리거: **Gmail → Watch Emails** (필터: has:attachment, 특정 발신자 또는 제목 키워드)
2. 액션 1: **Gmail → Get Attachments** (첨부파일 데이터 추출)
3. 액션 2: **Google Drive → Upload a File** (지정 폴더에 파일명 자동 지정 후 업로드)
4. 액션 3: **Notion → Create Database Item** (파일명, 발신자, 날짜, 드라이브 링크 자동 기록)

**포인트**: 이메일 필터를 잘 설정하는 것이 핵심입니다. "subject contains '인보이스'" 또는 "from:noreply@거래처도메인.com" 조건을 걸면 원하는 이메일만 골라서 처리할 수 있습니다.

> 💡 **실전 팁**: 세 시나리오를 모두 세팅하면 하루 소비 오퍼레이션은 약 40~70회 수준입니다(일 이메일 10건, 폼 5건, 스케줄 1회 기준). 무료 플랜(1,000회/월)으로도 커버되지만, 실수로 무한 루프 오류가 나면 순식간에 소진될 수 있습니다. 처음에는 "Maximum number of cycles"를 1~2로 낮게 설정해두고 안전하게 운영하세요.

## Make 초보자가 자주 빠지는 함정 5가지 — 이것만은 하지 마세요

실전에서 가장 많이 발생하는 실수들입니다. 미리 알고 피하면 삽질을 반나절 줄일 수 있어요.

### 함정 1: 트리거 테스트 없이 바로 스케줄 ON

새 시나리오를 만들고 테스트 없이 "Schedule" 스위치를 켜는 경우가 많습니다. 이렇게 하면 오류가 있는 시나리오가 계속 실행되면서 오퍼레이션을 소진합니다. **반드시 "Run once"로 먼저 테스트하고**, 각 모듈의 출력 데이터를 눈으로 확인한 뒤 스케줄을 켜세요.

### 함정 2: 모든 이메일에 트리거 걸기

Gmail 트리거를 "모든 수신 이메일"로 설정하면 뉴스레터, 광고 메일 포함 하루 수십~수백 건이 처리됩니다. 무료 플랜은 순식간에 오퍼레이션이 소진되고, 유료 플랜도 예상치 못한 과금이 발생할 수 있어요. **반드시 필터(발신자, 제목, 라벨 등)를 먼저 설정**하세요.

### 함정 3: 구글 연동 권한을 '제한된 액세스'로 설정

Make와 구글 계정을 연결할 때 "Google hasn't verified this app" 경고창이 뜹니다. 이걸 보고 "취소"를 누르거나 제한된 권한으로만 연결하면 나중에 Google Sheets 쓰기나 Gmail 발송이 안 되는 경우가 생깁니다. 개인 구글 계정이라면 "고급" → "안전하지 않은 앱으로 이동(안전)"을 눌러 전체 권한을 부여해야 합니다. (회사 구글 워크스페이스라면 관리자에게 앱 승인을 먼저 요청하세요.)

### 함정 4: 데이터 매핑에서 타입 불일치 오류 무시하기

Make에서 자주 발생하는 오류 중 하나가 숫자 필드에 텍스트 값이 들어가는 경우입니다. 예를 들어 구글 폼의 숫자 응답을 구글 시트의 숫자 형식 열에 넣을 때, Make가 그 값을 텍스트로 인식하면 시트에 입력은 되지만 SUM 함수가 작동하지 않습니다. **parseNumber()** 함수를 써서 명시적으로 형변환(타입 변환)하는 습관을 들이세요.

### 함정 5: 오류 알림을 꺼두기

시나리오가 오류로 멈춰도 모르고 며칠을 보내는 경우가 많습니다. Make 시나리오 설정에서 **"Send notifications" → 이메일 알림 ON**을 반드시 설정해두세요. 오류가 발생하는 즉시 이메일로 알림이 오기 때문에 빠르게 대응할 수 있습니다.

## Make와 함께 쓰면 더 강력해지는 노코드 자동화 도구 스택

Make 단독으로도 강력하지만, 아래 도구들과 조합하면 재택근무 생산성이 한 단계 더 올라갑니다.

### Make + 노션(Notion) 조합

노션은 데이터베이스 기능 덕분에 Make의 출력 데이터를 저장하는 '허브'로 최적입니다. Make → 노션으로 데이터를 보내는 것도 되고, 노션 DB 업데이트를 Make가 감지해 슬랙 알림을 보내는 것도 됩니다. 재택근무 프로젝트 관리, 클라이언트 CRM, 콘텐츠 캘린더 자동화에 특히 강합니다.

### Make + 구글 워크스페이스 조합

Gmail, Google Sheets, Google Drive, Google Calendar, Google Forms까지 Make와 구글의 연동은 가장 완성도가 높습니다. 재택근무 환경에서 구글 워크스페이스를 기본으로 쓴다면, 추가 도구 없이 Make + 구글만으로도 웬만한 자동화는 다 됩니다.

### Make + 슬랙(Slack) 조합

슬랙은 Make 자동화의 '출력 창구'로 가장 많이 쓰입니다. 자동화 결과를 슬랙 채널이나 DM으로 받으면 업무 흐름을 끊지 않고 정보를 받아볼 수 있어요. 반대로 슬랙 명령어(Slash Command)나 특정 채널의 메시지를 트리거로 Make 시나리오를 실행하는 것도 가능합니다.

> 💡 **실전 팁**: Make의 'Webhook' 모듈을 활용하면 Make가 공식 지원하지 않는 앱(카카오 API, 네이버 API, 사내 개발 시스템 등)도 연동할 수 있습니다. API 문서를 읽고 HTTP 요청을 직접 구성하는 방식으로, 코드를 몰라도 Make의 HTTP 모듈로 충분히 처리 가능합니다.

## ❓ 자주 묻는 질문

**Q1: Make 무료 플랜으로 실제 업무 자동화가 가능한가요?**

A1: 네, Make 무료 플랜(Free)은 월 1,000 오퍼레이션(작업 실행 횟수)과 최대 2개의 활성 시나리오를 제공합니다. 매일 이메일 1~2건 처리, 주간 리포트 자동 전송 정도의 가벼운 재택근무 자동화라면 무료 플랜으로도 충분히 운영할 수 있어요. 단, 스케줄 실행 주기가 최소 15분 간격으로 제한되고, 데이터 저장 용량도 제한됩니다. 본격적인 업무 자동화(하루 수십 건 이상 처리)를 원한다면 월 $9의 Core 플랜을 추천합니다. 2026년 4월 기준 Make 공식 사이트(make.com/en/pricing) 가격 기준입니다.

**Q2: Make랑 n8n 차이가 뭔가요? 어떤 걸 써야 하나요?**

A2: Make와 n8n의 가장 큰 차이는 '사용 편의성 vs 유연성'입니다. Make는 드래그앤드롭 방식의 시각적 빌더가 직관적이어서 코드 없이도 복잡한 자동화를 구성할 수 있어요. 반면 n8n은 자체 서버에 설치(self-hosted)할 수 있고, 커스텀 코드(JavaScript/Python) 삽입이 자유로워 개발자 친화적입니다. 재택근무자나 비개발자라면 Make가 훨씬 접근하기 쉽고, 연동 앱 수도 2026년 기준 1,700개 이상으로 n8n보다 많습니다. 개발 경험이 있고 서버를 직접 운영하고 싶다면 n8n이 유리합니다.

**Q3: Make(Integromat) 유료 플랜 가격이 얼마인가요? 가성비 있나요?**

A3: 2026년 4월 기준 Make의 요금제는 Free($0), Core($9/월), Pro($16/월), Teams($29/월)로 구성됩니다. 재택근무자 개인이라면 Core 플랜($9/월, 약 13,000원)이 가장 가성비가 좋아요. 월 10,000 오퍼레이션, 최소 1분 스케줄 간격, 무제한 활성 시나리오를 제공합니다. 이 정도면 이메일 자동화, 슬랙 알림, 구글 시트 데이터 처리 등 일반적인 재택근무 자동화를 모두 커버할 수 있습니다. 자포자기하고 매일 1시간씩 반복 업무에 쓰는 것보다 월 9달러 투자가 훨씬 합리적입니다.

**Q4: Make 자동화 설정하다가 오류 나면 어떻게 해결하나요?**

A4: Make에서 가장 자주 발생하는 오류는 세 가지입니다. 첫째, 연결(Connection) 인증 만료 — 구글/슬랙 등 앱을 다시 연결(Reconnect)하면 해결됩니다. 둘째, 오퍼레이션 초과 — 무료 플랜의 1,000회 제한을 넘으면 시나리오가 멈추므로 플랜 업그레이드가 필요합니다. 셋째, 데이터 매핑 오류 — 이전 모듈의 출력값이 다음 모듈의 입력 필드와 형식이 맞지 않을 때 발생하며, Make의 'Run Once' 기능으로 단계별 출력값을 확인해 디버깅할 수 있습니다. Make 공식 헬프센터(help.make.com)에 한국어 포함 다국어 가이드가 있습니다.

**Q5: Make로 자동화할 수 있는 앱이 카카오톡이나 네이버 연동도 되나요?**

A5: 2026년 4월 기준 Make는 카카오톡 공식 앱은 제공하지 않습니다. 다만 카카오 비즈니스 채널의 경우 Webhook 방식으로 연동이 가능하고, 카카오 API를 Make의 HTTP 모듈로 직접 호출하는 방법을 사용할 수 있습니다. 네이버 역시 공식 앱은 없지만 네이버 스마트스토어, 네이버 클라우드 등의 API를 Webhook이나 HTTP 모듈로 연동할 수 있어요. 국내 서비스 연동이 많다면 자피어(Zapier)보다 Make가, Make보다는 국내 특화 자동화 플랫폼인 '채널톡 자동화' 또는 n8n 커스텀 연동을 검토하는 것도 방법입니다.
