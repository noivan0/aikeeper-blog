## 🤔 Cursor AI, 도대체 뭐가 다른 건가요?

비개발자에게 AI 코딩 툴을 추천하면 가장 먼저 나오는 질문이 있습니다. "ChatGPT로 코드 만들어서 붙여 넣으면 되는 거 아닌가요?" 맞는 말이기도 하고, 완전히 틀린 말이기도 합니다.

### ChatGPT vs Cursor AI: 결정적 차이

ChatGPT로 코드를 생성하면 반드시 "복사 → 어딘가에 붙여넣기 → 오류 → 다시 ChatGPT에 오류 복사 → 수정 코드 받기 → 다시 붙여넣기"의 무한 루프를 겪게 됩니다. 개발 경험이 없는 분들은 이 과정에서 대부분 포기하죠.

Cursor AI는 이 문제를 근본적으로 해결했습니다. VS Code(세계에서 가장 많이 쓰이는 코드 편집기)를 기반으로 만들어진 Cursor는, AI가 편집기 **안에** 직접 들어와 있습니다. 코드를 ChatGPT에 복사할 필요 없이, Cursor 안에서 "이 버튼을 클릭하면 팝업이 뜨게 해줘"라고 말하면 AI가 직접 파일을 찾아 수정합니다.

| 항목 | ChatGPT + 복붙 | Cursor AI |
|------|----------------|-----------|
| 코드 생성 위치 | 외부 대화창 | 편집기 내부 |
| 오류 수정 방식 | 수동 복사 | 자동 감지 + 제안 |
| 파일 구조 이해 | 불가능 | 프로젝트 전체 이해 |
| 비개발자 난이도 | 높음 (붙여넣기 실수 多) | 중간 (대화만 하면 됨) |
| 배포 지원 | 없음 | 명령어 직접 생성 |

### 왜 지금 Cursor AI인가?

[Cursor AI 공식 통계](https://cursor.com)에 따르면 2025년 말 기준 월간 활성 사용자 수가 400만 명을 돌파했습니다. 특히 주목할 점은 전체 사용자의 약 35%가 비개발자(기획자, 마케터, 디자이너, 창업가)라는 점입니다. "Vibe Coding(바이브 코딩)"이라는 신조어까지 생겼는데, 이는 코드 문법보다 아이디어와 의도만으로 앱을 만드는 방식을 의미합니다.

2026년 4월 기준 Cursor AI는 Claude 3.7 Sonnet, GPT-4o, Gemini 1.5 Pro 등 최상위 AI 모델을 내장하고 있어, 비개발자가 자연어로 요청해도 실제로 동작하는 코드를 만들어낼 수 있습니다.

> 💡 **실전 팁**: Cursor AI를 처음 설치하면 AI 모델을 선택하라는 메뉴가 나옵니다. 비개발자라면 **Claude 3.7 Sonnet**을 기본 모델로 설정하세요. 코드 생성 품질도 높고, 한국어 요청을 가장 자연스럽게 이해합니다.

## 🛠️ 0단계: 설치부터 첫 화면까지 (30분이면 충분합니다)

본격적인 4단계 전에, Cursor AI를 처음 세팅하는 과정을 먼저 설명합니다. 이게 제일 막막한 부분이니까요.

### Cursor AI 설치 방법

1. [cursor.com](https://cursor.com)에 접속합니다.
2. "Download for Mac/Windows" 버튼을 클릭해 설치 파일을 받습니다.
3. 설치 후 실행하면 GitHub 또는 Google 계정으로 로그인하면 됩니다.
4. 모델 선택 화면에서 Claude 3.7 Sonnet을 선택합니다.
5. "New Project" → 빈 폴더 하나를 만들어 선택합니다.

전부 합해서 10분이면 끝납니다. 개발 환경 세팅에 며칠씩 걸리던 시대는 이미 끝났습니다.

### 처음 열었을 때 꼭 알아야 할 화면 구성

Cursor를 열면 왼쪽엔 파일 목록, 오른쪽엔 코드 편집 영역, 오른쪽 사이드바엔 AI 채팅창이 있습니다. 비개발자가 주로 사용할 부분은 딱 두 가지입니다.

- **Composer (단축키: Cmd/Ctrl + I)**: 새 파일을 만들거나 기존 파일을 대규모로 수정할 때. "로그인 페이지를 만들어줘"처럼 큰 요청에 씁니다.
- **Chat (단축키: Cmd/Ctrl + L)**: 질문하거나 작은 수정 요청을 할 때. "이 버튼이 왜 안 눌리는 거야?"처럼 대화하듯 씁니다.

> 💡 **실전 팁**: Cursor를 열고 Composer 창에 딱 이 한 줄만 입력해보세요. **"Hello World라고 화면에 보여주는 간단한 HTML 파일을 만들어줘."** AI가 파일을 만들어 주면, 그 파일을 더블클릭해 크롬 브라우저로 열어보세요. 여러분의 첫 웹페이지가 완성됩니다.

## 📋 1단계: 기획 — "만들 것"을 글로 정리하기

많은 분들이 Cursor AI를 켜자마자 막연하게 "웹앱 만들어줘"라고 입력합니다. 그러면 AI도 막연한 결과물을 냅니다. 4단계 중 가장 중요한 건 아이러니하게도 AI를 켜기 전, **기획을 글로 정리하는 것**입니다.

### 비개발자를 위한 기획 문서 작성법

개발자가 사용하는 기술 명세서(PRD)가 아니어도 됩니다. 다음 5가지 항목만 메모장에 적어보세요.

1. **이 웹앱은 누가 씁니까?** (예: 우리 팀 5명, 외부 고객, 나 혼자)
2. **이 웹앱에서 사용자가 하는 핵심 행동은?** (예: 이름 입력 후 결과 확인, 파일 업로드 후 다운로드)
3. **화면은 몇 개가 필요합니까?** (예: 메인 페이지 1개, 결과 페이지 1개)
4. **데이터를 저장해야 합니까?** (예: 아니오 / 예, 로컬에만 / 예, 서버에)
5. **어떤 느낌의 디자인을 원합니까?** (예: 깔끔한 흰색 배경, 네이버처럼, 노션처럼)

이 5가지 질문에 답하면 여러분만의 기획서가 완성됩니다.

### 기획을 프롬프트로 바꾸는 공식

기획 내용을 Cursor AI에게 전달하는 방법도 중요합니다. 아래 프롬프트 템플릿을 그대로 활용하세요.

```
나는 [누구]입니다. [무엇을 위한] 웹 앱을 만들고 싶습니다.
핵심 기능은 [기능 1], [기능 2], [기능 3]입니다.
화면은 [화면 수]개이고, 데이터는 [저장 여부]합니다.
디자인은 [원하는 느낌]으로 해주세요.
HTML, CSS, JavaScript만 사용해서 만들어주세요.
(데이터베이스 없이 작동하는 간단한 버전으로 먼저 만들어주세요.)
```

비개발자라면 처음엔 **"HTML, CSS, JavaScript만"**이라고 제한하는 것이 핵심입니다. 이 세 가지만으로 만든 앱은 배포가 가장 쉽고, 별도 서버 설정이 필요 없습니다.

> 💡 **실전 팁**: 좋아하는 웹사이트를 찾아서 "이 사이트처럼 생긴 디자인으로 만들어줘"라고 링크를 붙여 넣어도 됩니다. Cursor AI는 URL을 분석해 비슷한 레이아웃을 만들어 줄 수 있습니다.

## ⚙️ 2단계: 생성 — AI에게 첫 버전을 맡기기

기획서가 완성됐다면, 이제 Cursor AI에게 실제로 만들어달라고 요청할 차례입니다. 이 단계에서 많은 분들이 "첫 결과물이 별로면 어떡하지?"라는 불안을 느낍니다. 걱정 마세요. **첫 결과물은 항상 70점입니다.** 그리고 그게 정상입니다.

### Composer로 첫 버전 생성하기

Cmd/Ctrl + I를 눌러 Composer를 열고, 준비한 프롬프트를 붙여 넣습니다. AI가 생성하는 시간은 보통 30초~2분 정도입니다. 생성이 끝나면 화면 하단에 "Accept All" 버튼이 나타납니다. 클릭하면 코드가 파일로 저장됩니다.

생성된 파일(보통 index.html)을 오른쪽 클릭 → "Open in Browser"를 선택하면 크롬에서 결과물을 바로 확인할 수 있습니다.

### 첫 결과물을 보는 올바른 방법

비개발자가 가장 흔하게 하는 실수는 첫 결과물이 기대와 다르다고 포기하는 것입니다. 첫 버전은 "방향이 맞는지 확인하는 초안"으로 봐야 합니다. 체크해야 할 것은 세 가지입니다.

| 확인 항목 | 기대 수준 | 행동 방법 |
|-----------|-----------|-----------|
| 화면이 열리는가? | 반드시 Yes | No면 Cursor에 "열기가 안 돼, 오류 고쳐줘" |
| 원하는 화면 수가 있는가? | 대략 맞으면 OK | 없으면 "메뉴에 X 페이지 추가해줘" |
| 디자인이 완전히 다른가? | 다를 수 있음 | "노션처럼 깔끔하게 다시 디자인해줘" |

> 💡 **실전 팁**: 결과물이 마음에 안 들 때 "다시 만들어줘"보다는 **"어떤 부분이 마음에 안 드는지"를 구체적으로 말하세요.** "버튼이 너무 크고, 색이 빨간색인데 파란색으로 바꿔줘. 폰트도 좀 더 가늘게"처럼요. 구체적일수록 결과물이 빠르게 좋아집니다.

## 🔧 3단계: 수정 — 대화로 완성도를 높이기

2단계에서 첫 버전을 얻었다면, 이제 대화를 통해 완성도를 끌어올릴 차례입니다. 이 단계가 전체 과정에서 시간이 가장 많이 걸리지만, 동시에 가장 재미있는 단계입니다.

### 수정 요청 잘 하는 3가지 원칙

**원칙 1: 한 번에 하나씩만 요청하기**
"버튼 색도 바꾸고, 헤더도 고치고, 폰트도 바꾸고, 로그인 기능도 추가해줘"라고 한꺼번에 요청하면 AI가 충돌하는 변경사항을 만들어낼 확률이 높습니다. 하나씩 요청하고 확인하는 리듬을 유지하세요.

**원칙 2: 오류가 나면 그대로 붙여 넣기**
화면이 하얗게 되거나 "TypeError"같은 오류 메시지가 뜨면, 그 메시지를 복사해서 Cursor Chat에 그대로 붙여 넣으세요. "이 오류가 났어. 고쳐줘."라고만 하면 됩니다. 무슨 뜻인지 이해할 필요 없습니다.

**원칙 3: 원하는 결과물을 이미지로 보여주기**
말로 설명하기 어려운 디자인은 [Pinterest](https://pinterest.com)나 [Dribbble](https://dribbble.com)에서 마음에 드는 UI 스크린샷을 저장한 뒤 Cursor Chat에 업로드하고 "이런 스타일로 만들어줘"라고 하면 됩니다. 2026년 현재 Cursor AI는 이미지를 이해하는 멀티모달 기능을 지원합니다.

### 자주 쓰는 수정 요청 문장 모음

비개발자가 실제로 많이 쓰는 수정 요청 문장들을 정리했습니다.

```
- "메인 페이지의 배경색을 흰색으로 바꿔줘"
- "제목 폰트를 더 크고 굵게 해줘"
- "버튼에 마우스를 올리면 색이 바뀌게 해줘"
- "모바일에서도 잘 보이게 반응형으로 고쳐줘"
- "입력창에 아무것도 안 넣고 제출하면 경고 메시지를 보여줘"
- "결과가 나오는 부분을 박스로 감싸고 그림자를 추가해줘"
- "로딩 중임을 알려주는 스피너를 추가해줘"
```

> 💡 **실전 팁**: 수정을 10번 이상 반복했을 때 코드가 꼬이는 느낌이 든다면, Composer에서 **"현재 코드를 처음부터 정리해서 다시 작성해줘. 기능은 그대로 유지하되 코드를 깔끔하게 리팩토링해줘"**라고 요청하세요. AI가 누적된 복잡성을 정리해줍니다.

## 🚀 4단계: 배포 — 세상에 내 웹앱 공개하기

완성도가 80% 이상이 됐다면, 이제 실제 URL을 만들어 다른 사람이 접근할 수 있게 배포할 차례입니다. 비개발자들이 가장 두려워하는 단계이지만, 사실 가장 쉽습니다.

### Vercel로 5분 만에 배포하기

HTML/CSS/JavaScript로 만든 정적 웹앱이라면 Vercel이 가장 쉽습니다. 절차는 다음과 같습니다.

1. **GitHub 계정 만들기** (github.com, 무료, 5분 소요)
2. **GitHub에 코드 올리기**: Cursor의 터미널(Terminal) 창에 아래를 붙여 넣고, 어떻게 해야 하냐고 물어보세요. "GitHub에 내 코드를 올리고 싶어. 처음부터 알려줘."
3. **Vercel 연동**: vercel.com → "New Project" → GitHub 저장소 선택 → Deploy 버튼 클릭
4. **배포 완료**: 1~2분 후 `프로젝트명.vercel.app` 형태의 URL이 생성됩니다.

이 URL을 팀 슬랙에 붙여 넣으면, 개발자 없이 만든 여러분의 웹앱을 전 세계 누구나 접근할 수 있습니다.

### 배포 후 수정도 자동화됩니다

Vercel과 GitHub를 연동하면, 이후 Cursor에서 코드를 수정하고 GitHub에 올릴 때마다 Vercel이 자동으로 새 버전을 배포합니다. 이를 **CI/CD(자동 배포)**라고 하는데, 비개발자 입장에선 "수정하면 바로 반영된다"는 뜻입니다.

> 💡 **실전 팁**: 배포 후 URL을 Cursor Chat에 붙여 넣고 "이 URL에서 발생할 수 있는 보안 문제나 성능 문제가 있으면 알려줘"라고 물어보세요. AI가 체크리스트를 제공합니다. 무료 서비스인 만큼 개인정보 수집은 최소화하는 것을 기본 원칙으로 삼으세요.

## ⚠️ 초보가 절대 빠지면 안 되는 함정 5가지

Cursor AI가 마법같이 보이지만, 비개발자가 흔히 저지르는 실수들이 있습니다. 미리 알면 시행착오를 절반으로 줄일 수 있습니다.

### 함정 1: 너무 복잡한 첫 프로젝트를 고르는 것
"로그인·결제·관리자·알림까지 다 되는 쇼핑몰"을 첫 프로젝트로 잡으면 100% 좌절합니다. 첫 프로젝트는 반드시 **화면 1~2개, 기능 2~3개** 수준으로 시작하세요.

### 함정 2: "Accept All"을 무작정 누르는 것
Cursor가 변경 사항을 제안할 때 항상 **미리보기(Preview)**를 먼저 확인하세요. 특히 여러 파일을 한 번에 수정하는 경우, 의도치 않은 파일이 변경될 수 있습니다.

### 함정 3: 작업 중간에 저장을 안 하는 것
Cursor는 자동 저장이 기본이지만, 중요한 단계를 마칠 때마다 GitHub에 커밋(저장 기록)을 남기세요. "현재 상태를 GitHub에 저장하는 명령어를 알려줘"라고 Cursor에게 물어보면 됩니다.

### 함정 4: AI가 틀릴 수 없다고 믿는 것
Cursor AI도 가끔 존재하지 않는 함수를 만들거나, 오래된 방법으로 코드를 작성합니다. 결과물이 이상하다 싶으면 "이 방법이 2026년 현재 최신 방법이 맞아? 더 나은 방법이 있으면 알려줘"라고 직접 물어보세요.

### 함정 5: 개인정보를 코드 안에 넣는 것
API 키, 비밀번호, 개인정보가 담긴 파일을 Cursor AI에게 보여주거나 GitHub에 올리는 실수를 하면 보안 문제가 생깁니다. API 키는 반드시 `.env` 파일로 분리해 관리하고, `.gitignore`에 추가하세요. Cursor AI에게 "API 키를 안전하게 관리하는 방법을 알려줘"라고 물어보면 됩니다.

## ❓ 자주 묻는 질문

**Q1: Cursor AI 완전 무료로 쓸 수 있나요?**
A1: Cursor AI는 2026년 4월 기준 무료 플랜(Hobby)과 유료 플랜(Pro, $20/월)을 제공합니다. 무료 플랜에서도 GPT-4o, Claude 3.7 Sonnet 등 고성능 모델을 월 200회까지 사용할 수 있어 비개발자가 첫 웹앱을 만들기에는 충분합니다. 다만 무료 플랜은 AI 자동완성(Tab) 기능이 하루 2,000회로 제한되고, Composer 사용 횟수도 제한됩니다. 본격적으로 프로젝트를 진행하거나 매일 사용한다면 Pro 플랜이 훨씬 효율적입니다. 처음엔 무료로 시작해 익숙해진 뒤 업그레이드하는 전략을 추천합니다.

**Q2: Cursor AI랑 ChatGPT랑 뭐가 달라요?**
A2: ChatGPT는 대화창에서 코드를 생성해 주는 반면, Cursor AI는 실제 코드 편집기(IDE) 안에 AI가 내장된 툴입니다. 가장 큰 차이는 "코드를 복사해서 붙여넣기" 할 필요가 없다는 점이에요. Cursor AI는 내 프로젝트 파일 전체를 이해하고, 수정이 필요한 코드를 직접 편집하거나 새 파일을 생성합니다. 또한 에러가 발생하면 Cursor가 자동으로 오류를 읽고 수정 방법을 제안하는 인라인 디버깅 기능도 있습니다. 비개발자 입장에서는 Cursor가 ChatGPT보다 훨씬 "실제 결과물"을 만드는 데 가깝습니다.

**Q3: 비개발자인데 Cursor AI로 정말 웹앱을 만들 수 있나요?**
A3: 네, 실제로 가능합니다. 2025~2026년 사이 전 세계적으로 비개발자·기획자·마케터들이 Cursor AI를 이용해 사내 툴, 포트폴리오 사이트, 간단한 SaaS 대시보드를 만든 사례가 급증했습니다. 핵심은 "코드를 이해하려고 하지 말고, 원하는 것을 글로 설명하는 능력"입니다. Cursor AI에게 "로그인 버튼을 파란색으로 바꿔줘", "사용자가 이름을 입력하면 환영 메시지를 보여주는 폼을 만들어줘"처럼 일상 언어로 요청하면 됩니다. 물론 복잡한 기업용 시스템을 혼자 만들기엔 한계가 있지만, MVP(최소기능제품) 수준의 웹앱은 충분히 가능합니다.

**Q4: Cursor AI로 만든 웹앱을 무료로 배포할 수 있나요?**
A4: 네, Vercel과 Netlify 두 서비스를 이용하면 무료로 배포할 수 있습니다. Vercel은 Next.js 기반 프로젝트에, Netlify는 정적 사이트나 React 앱에 주로 사용됩니다. 두 서비스 모두 GitHub 계정과 연동해 코드를 올리면 자동으로 웹에 배포해 주는 방식입니다. 무료 플랜 기준으로 커스텀 도메인 연결, HTTPS 자동 적용, 월 100GB 트래픽까지 제공됩니다. Cursor AI 내에서 Composer로 배포 명령어를 생성해 달라고 요청하면, 비개발자도 터미널 명령 한 줄로 배포할 수 있습니다.

**Q5: Cursor AI 한국어로 사용할 수 있나요?**
A5: Cursor AI 인터페이스 자체는 영어로 제공되지만, AI와의 대화(Composer, Chat)는 한국어로 완벽하게 작동합니다. "로그인 폼을 만들어줘", "버튼 색상을 초록색으로 바꿔줘"처럼 한국어로 요청하면 AI가 이해하고 코드를 생성합니다. 다만 코드 자체는 영어 기반으로 생성되는데, 이는 전 세계 프로그래밍 표준이기 때문에 정상입니다. UI 텍스트(버튼 이름, 안내문 등)는 한국어로 넣어달라고 요청하면 됩니다. 2026년 4월 기준, 한국 사용자 커뮤니티도 매우 활발하게 운영 중이어서 막히는 부분은 검색으로 쉽게 해결할 수 있습니다.

**Cursor AI 사용법**은 하루 이틀이면 익힐 수 있습니다. 중요한 건 기술이 아니라 **"내가 뭘 만들고 싶은가"를 명확히 아는 것**이고, 이건 비개발자인 여러분이 더 잘 알 수 있는 영역입니다. 기획, 사용자 이해, 문제 정의. 이게 결국 좋은 웹앱을 만드는 핵심 능력이거든요.

오늘 당장 cursor.com에 접속해서 설치하고, 딱 한 줄만 입력해보세요.

> "나를 위한 To-Do 리스트 웹앱을 만들어줘. 할 일을 추가하고 완료 표시를 할 수 있어야 해. 깔끔한 흰색 디자인으로."
