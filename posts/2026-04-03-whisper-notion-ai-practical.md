# Whisper는 이미 쓰고 있는데, 거기서 멈추는 사람이 90%다

인터뷰도 마찬가지입니다. 30분짜리 현장 인터뷰를 Whisper로 변환해놓고, 그 txt 파일을 그냥 폴더에 쌓아둡니다. 누가 나중에 "그 인터뷰에서 뭐라고 했어요?"라고 물으면 파일을 열어서 Ctrl+F로 검색하는 게 전부예요. 이게 지금 여러분이 하고 있는 일인가요?

**Whisper + Notion AI 연동**을 제대로 세팅하면 이 흐름이 완전히 바뀝니다. 음성 파일 하나를 올리면 → 텍스트 변환 → AI 요약 → Notion 데이터베이스 자동 저장까지 이어지는 파이프라인이 만들어지거든요. 연구자·프리랜서라면 이 세팅 하나로 주당 5~10시간의 정리 시간을 줄일 수 있습니다. 지금부터 실제로 작동하는 세팅을 처음부터 끝까지 알려드릴게요.

## 🔍 Whisper와 Notion AI, 각자의 역할부터 이해하자

Whisper + Notion AI 연동을 처음 시도하는 분들이 가장 많이 하는 실수가 있습니다. 두 도구가 "하나의 제품처럼" 바로 연결될 거라고 기대하는 거예요. 실제로는 그렇지 않습니다. 각각의 역할을 명확히 이해해야 연동 세팅이 흔들리지 않아요.

### Whisper: 음성을 텍스트로 바꾸는 변환기

[OpenAI의 Whisper](https://openai.com/research/whisper)는 2022년 공개된 오픈소스 음성 인식 모델입니다. 2026년 현재 large-v3 버전이 주력으로 쓰이며, 한국어를 포함한 99개 언어를 지원합니다. 특징은 크게 두 가지예요.

**첫째, 정확도가 높습니다.** 한국어 단어 오류율(WER)은 조용한 환경 기준 약 8~12% 수준으로, 네이버 클로바나 카카오 음성 API와 비교해도 전문 용어·고유명사 인식에서 강점을 보입니다.

**둘째, 사용 방식이 두 가지로 나뉩니다.** API 방식(유료, $0.006/분)과 로컬 실행 방식(무료, GPU 필요)으로 나뉘는데, 실용적인 선택 기준은 단순합니다. 월 처리량이 200분 미만이면 API, 그 이상이면 로컬 실행이 경제적입니다.

### Notion AI: 텍스트를 '구조화된 지식'으로 만드는 편집자

Notion AI는 단순한 텍스트 생성 도구가 아닙니다. 2025년 업데이트 이후 Notion AI는 데이터베이스 연동 기반의 **자동 분류·요약·태그 생성** 기능이 크게 강화됐어요. Whisper가 뽑아낸 날 것의 텍스트를 Notion AI가 받아서 다음을 수행합니다:

- **요약**: 1시간 강의 스크립트를 5줄 핵심 요약으로
- **태그 자동 생성**: 주제 키워드 추출 (예: "머신러닝", "2026 트렌드", "실습")
- **섹션 분류**: 인터뷰라면 Q&A 형식으로, 강의라면 챕터별로 자동 분리
- **액션 아이템 추출**: "다음 주까지 ~해야 한다" 같은 문장을 To-Do로 자동 분리

> 💡 **실전 팁**: Notion AI에 스크립트를 붙여넣을 때 "이 텍스트는 [인터뷰/강의] 녹취록입니다. 다음 형식으로 정리해주세요: 1) 3문장 요약, 2) 핵심 키워드 5개, 3) 주요 발언 Top 3"처럼 역할을 명시하면 결과물 품질이 2배 이상 좋아집니다.

## 🔧 전체 워크플로우 설계: 3단계 파이프라인

실제로 작동하는 Whisper + Notion 연동 시스템은 3개의 레이어로 구성됩니다. 이 구조를 머릿속에 먼저 그리고 나서 세팅을 시작하세요.

### 레이어 1 — 입력 (Input): 음성 파일 수집

음성 파일의 출처는 크게 세 가지입니다:

| 출처 | 포맷 | 권장 처리 방식 |
|------|------|--------------|
| 현장 인터뷰 녹음 | mp3, m4a, wav | Whisper API |
| 줌(Zoom) 강의 녹화 | mp4 | 영상→음성 추출 후 Whisper |
| 팟캐스트·유튜브 참고 자료 | 유튜브 URL | yt-dlp로 오디오 추출 |

파일 포맷은 mp3가 용량 대비 정확도 면에서 가장 효율적입니다. wav는 정확도가 약간 높지만 파일 크기가 mp3의 5~6배라 API 비용·처리 시간이 늘어납니다.

### 레이어 2 — 변환 (Transform): Whisper 실행

변환 단계에서 핵심은 **청크(Chunk) 분할**입니다. Whisper API는 파일 하나의 최대 크기를 25MB로 제한합니다. 60분 mp3는 보통 50~60MB라 그냥 올리면 오류가 납니다. 해결책은 두 가지예요.

- **방법 A**: ffmpeg로 25분 단위로 파일을 자르고 각각 처리
- **방법 B**: 로컬 Whisper를 쓰면 파일 크기 제한 없음

### 레이어 3 — 저장 (Store): Notion DB 자동 입력

변환된 텍스트를 Notion에 저장할 때 그냥 페이지에 붙여넣는 게 아니라 **데이터베이스 속성으로 구조화**해야 합니다. 권장 DB 구조는 이렇습니다:

| 속성명 | 타입 | 내용 |
|--------|------|------|
| 제목 | Title | 강의명/인터뷰 대상자명 |
| 날짜 | Date | 녹음 날짜 |
| 유형 | Select | 강의/인터뷰/회의/기타 |
| 태그 | Multi-select | AI 자동 생성 키워드 |
| 요약 | Text | Notion AI 3문장 요약 |
| 원본 스크립트 | Text | Whisper 전체 변환 텍스트 |
| 처리 상태 | Status | 변환완료/AI정리완료/검토완료 |

> 💡 **실전 팁**: "처리 상태" 속성을 활용해 Notion 필터를 걸면, 아직 AI 정리가 안 된 스크립트만 뽑아서 일괄 처리할 수 있습니다. 매주 금요일에 "AI정리완료" 아닌 항목을 필터링해 30분 안에 한 주 분량을 정리하는 루틴을 만들어두세요.

## 🛠️ 노코드 세팅: Make(메이크)로 30분 만에 자동화하기

코딩이 부담스럽다면 [Make(메이크)](https://www.make.com)를 활용하세요. 2025년 하반기부터 Make에 Whisper 공식 모듈이 추가되어 드래그앤드롭으로 연동이 가능합니다.

### Make 시나리오 구성 순서

**Step 1. 트리거 설정**
Google Drive나 Dropbox에 음성 파일이 업로드되면 자동으로 시나리오가 실행되도록 트리거를 설정합니다. "Watch Files in Folder" 모듈을 선택하고 지정 폴더(예: `/강의녹음/2026`)를 연결하세요.

**Step 2. Whisper 모듈 연결**
Make의 OpenAI 모듈에서 "Create a Transcription (Whisper)" 선택 → API 키 입력 → 언어를 "ko"(한국어)로 설정합니다. 이때 `response_format`을 `srt`로 설정하면 타임스탬프도 함께 저장되어 나중에 특정 발언 위치를 찾기가 훨씬 쉽습니다.

**Step 3. GPT-4o 요약 모듈 추가**
Whisper 출력 텍스트를 GPT-4o 모듈로 넘깁니다. 이때 사용하는 시스템 프롬프트 예시:
```
당신은 학술 연구자의 녹취록을 정리하는 전문 편집자입니다.
아래 녹취록을 다음 형식으로 정리하세요:
1. 핵심 요약 (3문장 이내)
2. 주요 키워드 (5개, #태그 형식)
3. 중요 발언 Top 3 (원문 인용)
4. 후속 액션 아이템 (있을 경우)
```

**Step 4. Notion 모듈로 DB 자동 저장**
GPT-4o 출력값을 파싱해서 각각 Notion DB 속성에 매핑합니다. 제목 → 파일명, 날짜 → 업로드 날짜, 요약 → GPT 요약, 원본 → Whisper 전체 텍스트.

**Make 플랜별 처리 가능 용량 비교:**

| 플랜 | 월 작업 수 | 가격 | 적합 대상 |
|------|----------|------|----------|
| Free | 1,000 | 무료 | 테스트용 |
| Core | 10,000 | $9/월 | 개인 연구자 |
| Pro | 100,000 | $16/월 | 프리랜서·소규모 팀 |

> 💡 **실전 팁**: Make 무료 플랜으로 먼저 시나리오를 만들어 테스트하세요. 30분짜리 인터뷰 1개 처리에 약 15~20 operations가 소요됩니다. 월 10개 이하 처리라면 무료 플랜으로도 충분합니다.

## 💻 Python 세팅: 더 정밀한 제어가 필요할 때

연구자나 데이터가 많은 프리랜서라면 Python 스크립트로 직접 파이프라인을 구축하는 게 장기적으로 훨씬 강력합니다. 비용도 절감되고, 커스터마이징 폭도 넓어요.

### 필수 라이브러리 설치

```bash
pip install openai notion-client pydub python-dotenv
```

### 핵심 코드 구조

```python
import openai
from notion_client import Client
from pydub import AudioSegment
import os
from dotenv import load_dotenv

load_dotenv()

# API 클라이언트 초기화
openai_client = openai.OpenAI(api_key=os.getenv("OPENAI_API_KEY"))
notion = Client(auth=os.getenv("NOTION_TOKEN"))

def transcribe_audio(file_path: str) -> str:
    """25MB 초과 파일은 자동으로 청크 분할 처리"""
    file_size = os.path.getsize(file_path)
    
    if file_size > 25 * 1024 * 1024:  # 25MB 초과
        return transcribe_long_audio(file_path)
    
    with open(file_path, "rb") as audio_file:
        transcript = openai_client.audio.transcriptions.create(
            model="whisper-1",
            file=audio_file,
            language="ko",
            response_format="text"
        )
    return transcript

def summarize_with_gpt(transcript: str) -> dict:
    """GPT-4o로 요약·태그·액션아이템 추출"""
    response = openai_client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {"role": "system", "content": "녹취록 정리 전문가입니다. JSON 형식으로만 응답하세요."},
            {"role": "user", "content": f"""
            다음 녹취록을 분석해 JSON으로 반환하세요:
            {{
              "summary": "3문장 요약",
              "keywords": ["키워드1", "키워드2", ...],
              "key_quotes": ["중요 발언1", ...],
              "action_items": ["액션아이템1", ...]
            }}
            
            녹취록: {transcript[:4000]}
            """}
        ]
    )
    import json
    return json.loads(response.choices[0].message.content)

def save_to_notion(title: str, transcript: str, summary_data: dict, date: str):
    """Notion DB에 페이지 생성"""
    notion.pages.create(
        parent={"database_id": os.getenv("NOTION_DATABASE_ID")},
        properties={
            "제목": {"title": [{"text": {"content": title}}]},
            "날짜": {"date": {"start": date}},
            "요약": {"rich_text": [{"text": {"content": summary_data["summary"]}}]},
            "태그": {"multi_select": [{"name": kw} for kw in summary_data["keywords"]]},
            "처리 상태": {"status": {"name": "AI정리완료"}}
        },
        children=[
            {
                "object": "block",
                "type": "paragraph",
                "paragraph": {
                    "rich_text": [{"type": "text", "text": {"content": transcript[:2000]}}]
                }
            }
        ]
    )
```

## ⚠️ 주의사항: 이 함정은 꼭 피하세요

실제로 이 세팅을 쓰면서 가장 많이 걸리는 실수들입니다. 미리 알면 삽질을 피할 수 있어요.

### 함정 1: Whisper API 파일 크기 제한 무시

앞서 언급했지만 가장 많이 걸리는 오류입니다. 25MB 초과 파일을 그냥 올리면 `413 Request Entity Too Large` 오류가 납니다. 반드시 ffmpeg나 pydub으로 사전 분할하거나, 로컬 Whisper를 사용하세요.

### 함정 2: 언어 파라미터 미설정

Whisper는 언어를 자동 감지하는 기능이 있는데, 한국어 강의에서 영어 단어가 자주 등장하면 언어 감지가 흔들려서 정확도가 떨어집니다. 반드시 `language="ko"`를 명시하세요. 영어·한국어 혼용 강의라면 `language=None`으로 auto로 두되, 결과물을 한 번 검토하는 루틴을 만드세요.

### 함정 3: Notion API 텍스트 블록 2000자 제한

Notion API는 하나의 텍스트 블록에 2000자까지만 입력됩니다. 60분 강의 스크립트는 보통 8,000~15,000자인데, 이걸 한 번에 넣으려 하면 데이터가 잘립니다. Python 코드에서 2000자 단위로 분할해 여러 블록으로 나눠 저장하는 로직이 반드시 필요합니다.

### 함정 4: Notion AI 요약의 환각(Hallucination) 미검토

Notion AI가 요약한 내용이 원본 스크립트에 없는 내용을 추가하는 경우가 간혹 있습니다. 특히 전문 용어가 많은 학술 인터뷰나 기술 강의에서 발생 빈도가 높습니다. 요약본을 최종 자료로 쓰기 전에 원본 스크립트와 대조 검토하는 단계를 반드시 유지하세요. 이 파이프라인은 "초안 자동 생성"이지 "완성본 자동 생성"이 아닙니다.

### 함정 5: 개인정보 동의 없이 API 전송

인터뷰 대상자의 음성을 OpenAI API 서버로 전송하기 전에 반드시 서면 동의를 받아야 합니다. 특히 의료·법률·금융 분야 인터뷰는 더 엄격한 동의 절차가 필요합니다. 동의 없이 처리할 경우 개인정보보호법 위반이 될 수 있어요. 민감한 인터뷰는 반드시 로컬 Whisper(오프라인)를 사용하세요.

## ❓ 자주 묻는 질문

**Q1: Whisper로 변환한 텍스트를 Notion에 자동으로 저장할 수 있나요?**

A1: 네, 가능합니다. Whisper API로 음성을 텍스트로 변환한 후, Notion API(또는 Make·Zapier 같은 자동화 도구)를 연결하면 변환된 스크립트가 지정한 Notion 데이터베이스에 자동 저장됩니다. Python 스크립트 한 줄로도 구현할 수 있고, 코딩이 어려운 분은 Make(구 Integromat)의 Whisper→Notion 템플릿을 사용하면 노코드로도 세팅 가능합니다. 2026년 현재 Make에는 Whisper 모듈이 공식 지원되어 별도 API 설정 없이 드래그앤드롭으로 연결할 수 있습니다.

**Q2: Whisper 한국어 인식 정확도가 실제로 어떤가요?**

A2: OpenAI Whisper large-v3 모델 기준, 한국어 단어 오류율(WER)은 약 8~12% 수준입니다. 표준 발음과 조용한 환경에서는 95% 이상의 정확도를 보이지만, 사투리·전문 용어·잡음이 많은 환경에서는 오류가 늘어납니다. 실무에서는 Notion AI의 교정 기능을 연결해 전문 용어 오탈자를 자동 수정하거나, 자주 나오는 전문 용어를 프롬프트 앞에 힌트로 제공하는 방식으로 정확도를 90% 이상으로 끌어올릴 수 있습니다.

**Q3: 강의 녹음 파일을 Whisper로 변환하면 얼마나 걸리나요?**

A3: Whisper API 기준, 60분 강의 파일(mp3, 약 50MB)을 25분 단위로 분할하면 각 청크당 1~2분, 전체 2~4분 안에 변환이 완료됩니다. 로컬에서 Whisper를 직접 실행할 경우 GPU 성능에 따라 다르지만 NVIDIA RTX 3080 환경에서 60분 파일을 약 5~8분 안에 처리할 수 있습니다. 맥북 M3 Pro 기준으로는 6~9분 정도 소요됩니다. 무료로 사용하려면 로컬 실행이 낫고, 빠르고 안정적인 처리가 필요하다면 API($0.006/분)를 추천합니다.

**Q4: Whisper Notion 연동할 때 개인정보 보호는 어떻게 하나요?**

A4: 인터뷰 대상자의 음성이나 민감한 강의 내용을 다룰 때는 OpenAI API로 전송하기 전 반드시 동의를 받아야 합니다. 개인정보가 우려된다면 Whisper를 로컬 환경(오프라인)에서 실행하는 것이 가장 안전합니다. Notion 쪽에서는 워크스페이스를 팀 공유가 아닌 개인 전용으로 설정하고, 민감 페이지에 비공개 잠금을 걸어두는 것을 권장합니다. 기업 환경이라면 Notion Enterprise + 자체 호스팅 Whisper 조합을 고려하세요. 의료·법률 분야는 관련 법규(의료법, 개인정보보호법)를 사전에 반드시 확인하세요.

**Q5: 코딩 못해도 Whisper + Notion 연동 세팅할 수 있나요?**

A5: 충분히 가능합니다. Make(메이크, 구 Integromat)나 Zapier를 사용하면 코드 한 줄 없이 음성 파일 업로드→Whisper 변환→Notion 저장 자동화를 구축할 수 있습니다. 특히 Make의 경우 2025년 하반기부터 Whisper 모듈을 공식 지원하면서 드래그앤드롭으로 워크플로우를 완성할 수 있게 됐습니다. Make Core 플랜($9/월)으로 월 10,000 operations를 쓸 수 있어 개인 연구자·프리랜서에게는 충분한 용량입니다. 이 글 본문의 Make 세팅 섹션을 참고하시면 30분 안에 세팅을 마칠 수 있습니다.

## 마무리: 이제 Whisper에서 멈추지 마세요

Whisper로 텍스트 변환까지만 쓰는 건, 강력한 주방 칼을 사서 포장만 뜯어보고 서랍에 넣어두는 것과 같습니다. 실제 가치는 Notion AI와 연결해서 '검색 가능한 지식 베이스'를 쌓을 때 비로소 생기거든요.

이 세팅이 처음엔 다소 복잡하게 느껴질 수 있습니다. 하지만 한 번 구축해두면 이후에는 음성 파일을 폴더에 넣는 것만으로 요약·태그·DB 저장이 자동으로 이뤄집니다. 연구자라면 인터뷰 코딩 시간을, 프리랜서라면 아티클 초안 작성 시간을 절반 이하로 줄일 수 있어요.

2026년 현재, [OpenAI의 Whisper API](https://platform.openai.com/docs/guides/speech-to-text)는 분당 $0.006으로 운영 중이고, Notion AI는 Notion Plus 플랜($10/월)에 포함되어 있습니다. 둘 합쳐서 월 2~3만 원 수준이면 주당 5~10시간을 되찾을 수 있습니다.

**여러분에게 질문 드릴게요.** 지금 가장 어려운 단계가 어디인가요? Make 세팅이 막히나요? Python 코드 중 오류가 나나요? 아니면 Notion DB 구조를 어떻게 짜야 할지 모르겠나요? 댓글로 구체적인 상황을 남겨주시면 직접 답변드리겠습니다. 다음 글에서는 이 파이프라인을 **Obsidian**과 연결해 로컬 기반 완전 오프라인 지식 베이스를 만드는 세팅도 다룰 예정이에요.
