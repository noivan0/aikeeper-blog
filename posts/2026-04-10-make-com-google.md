## Make.com이 n8n보다 쉬운 이유: 브라우저만으로 즉시 시작

n8n을 처음 써보려고 공식 문서를 열었을 때의 그 당혹감, 기억하시나요? Docker 설치, 포트 포워딩, 환경변수 설정… 자동화보다 서버 공부를 먼저 해야 할 것 같은 느낌이요. [n8n 공식 문서](https://docs.n8n.io/hosting/)에도 셀프 호스팅 설정이 첫 페이지를 차지합니다.

Make.com(구 Integromat)은 다릅니다. 회원가입 후 로그인하면 그게 끝입니다. 설치 없음, 서버 없음, 코드 없음.

### Make.com의 핵심 차별점

Make.com은 2022년 Integromat에서 리브랜딩한 노코드 자동화 플랫폼입니다. 2026년 4월 기준 200개국 이상에서 500,000개 이상의 조직이 사용하는 것으로 알려져 있습니다 (출처: Make.com 공식 소개 페이지). 핵심 UI는 '원형 모듈을 연결하는 시각적 워크플로우 빌더'로, 각 앱·서비스를 동그란 아이콘으로 표현하고 화살표로 연결하는 방식입니다.

워크플로우를 '시나리오(Scenario)'라고 부르며, 하나의 시나리오 안에 모듈(Module)을 연결해 데이터 흐름을 설계합니다. 구글 시트, 슬랙, Gmail, Notion, Airtable 등 1,000개 이상의 앱과 연동이 가능합니다.

### n8n · Zapier · Make.com 3파전 비교

n8n이 어려웠던 분, Zapier가 너무 비쌌던 분 모두에게 Make.com이 현실적인 대안입니다.

> 💡 **실전 팁**: Make.com은 무료 플랜에서도 '즉시 실행(Run once)' 기능으로 시나리오를 테스트할 수 있습니다. 유료 전환 없이 먼저 구조를 완성하고 테스트한 뒤 활성화하세요.

> 💡 **실전 팁**: 오퍼레이션 소비량은 Make.com 대시보드의 'Usage' 탭에서 실시간으로 확인할 수 있습니다. 무료 플랜 한도에 가까워지면 이메일 알림도 발송됩니다.

## Make 워크플로우 만들기: 구글 시트→슬랙 단계별 실전 과정

드디어 본론입니다. 직접 따라 해보시면 약 25~30분이면 완성됩니다. 각 단계를 순서대로 진행하세요.

### 단계 1: 새 시나리오(Scenario) 만들기

1. Make.com에 로그인 후 왼쪽 사이드바에서 **Scenarios** 클릭
2. 우측 상단 **+ Create a new scenario** 버튼 클릭
3. 빈 캔버스 화면이 열리면, 가운데 큰 **+** 버튼을 클릭해 첫 번째 모듈 추가

검색창에 **"Google Sheets"** 입력 후 선택합니다.

### 단계 2: 구글 시트 트리거 모듈 설정

Google Sheets 모듈 선택 후, 아래 옵션 중 상황에 맞게 선택합니다.

| 트리거 유형 | 설명 | 추천 상황 |
|------------|------|----------|
| Watch Rows | 새 행이 추가될 때마다 실행 | 실시간 데이터 입력 보고 |
| Watch Changes | 셀 값이 변경될 때 실행 | 상태 업데이트 모니터링 |
| Search Rows | 조건에 맞는 행을 검색 | 정기 집계 보고서 |
| Get a Range of Values | 특정 범위를 통째로 가져옴 | 주간·월간 요약 보고서 |

이 튜토리얼에서는 **"Watch Rows"** (새 행 추가 시 실행)를 선택합니다.

설정 항목:
- **Connection**: 구글 계정 연결 (처음이면 'Add' 클릭 → 구글 로그인 팝업 → 허용)
- **Spreadsheet ID**: 보고서로 보낼 구글 시트를 선택 (드롭다운에서 선택 가능)
- **Sheet Name**: 시트 탭 이름 선택 (예: `Sheet1` 또는 `데이터`)
- **Table contains headers**: Yes 선택 (1행이 헤더인 경우)
- **Where to start**: 처음 실행 시 어느 행부터 볼지 설정 (`From all` → 기존 데이터 전체, `From now on` → 이후 추가되는 데이터만)

> 💡 **실전 팁**: 처음 설정할 때는 반드시 'Where to start'를 **'From now on'**으로 설정하세요. 'From all'로 놓으면 기존 데이터 수백 행이 한꺼번에 슬랙으로 발송되는 참사가 발생할 수 있습니다.

### 단계 3: 슬랙 메시지 발송 모듈 추가

구글 시트 모듈 오른쪽에 있는 **+** 버튼을 클릭해 두 번째 모듈을 추가합니다.

검색창에 **"Slack"** 입력 → **Create a Message** 선택

설정 항목:
- **Connection**: 슬랙 워크스페이스 연결 ('Add' → 슬랙 로그인 → 권한 허용)
- **Channel**: 메시지를 보낼 채널 선택 (예: `#daily-report`)
- **Text**: 슬랙에 표시될 메시지 내용 입력

**Text 필드**가 핵심입니다. 여기서 구글 시트의 데이터를 동적으로 삽입합니다. 텍스트 필드를 클릭하면 오른쪽에 '사용 가능한 변수' 패널이 열립니다. 구글 시트 모듈에서 가져온 각 컬럼(날짜, 담당자, 항목 등)이 변수로 표시됩니다.

아래처럼 메시지를 구성해보세요:

```
📊 *새 데이터가 입력되었습니다*

📅 날짜: {{1.날짜}}
👤 담당자: {{1.담당자}}
📦 항목: {{1.항목}}
🔢 수량: {{1.수량}}개
💰 매출: {{1.매출(원)}}원
```

`{{1.컬럼명}}` 형태가 Make.com의 동적 변수입니다. 숫자 `1`은 첫 번째 모듈(구글 시트)을 의미합니다.

> 💡 **실전 팁**: 슬랙에서 `*텍스트*`는 굵게, `_텍스트_`는 기울임, `:이모지이름:`은 이모지로 표시됩니다. 메시지에 이모지를 적절히 활용하면 가독성이 크게 높아집니다.

### 단계 4: 시나리오 테스트 실행 (Run once)

모듈 연결이 완료됐다면, 화면 하단의 **Run once** 버튼을 클릭합니다. Make.com이 구글 시트에서 데이터를 읽어 슬랙으로 메시지를 발송합니다. 각 모듈 위에 녹색 숫자(처리된 데이터 개수)가 표시되면 성공입니다.

슬랙 채널을 열어 메시지가 정상적으로 수신됐는지 확인하세요.

### 단계 5: 정기 실행 스케줄링

테스트가 성공했다면, 이제 언제 자동으로 실행할지 설정합니다.

화면 하단의 **Scheduling** 토글을 켭니다. 기어 아이콘(⚙️)을 클릭하면 실행 주기를 설정할 수 있습니다.

| 실행 주기 | 설정 방법 | 적합한 사용 사례 |
|-----------|----------|----------------|
| 매 5분 | Every 5 minutes | 실시간 알림 (무료 플랜 최소 단위) |
| 매 1분 | Every 1 minute | 긴급 알림 (Core 이상 플랜) |
| 매일 오전 9시 | Daily, 09:00 | 일일 업무 시작 보고서 |
| 매주 월요일 | Weekly, Monday | 주간 실적 요약 |
| 매월 1일 | Monthly, 1st | 월간 보고서 |

> 💡 **실전 팁**: 무료 플랜의 최소 실행 간격은 15분입니다 (2026년 4월 기준). 1분 단위 실행이 필요하다면 Core 플랜($9/월)이 필요합니다.

설정 완료 후 **Save** 버튼을 누르고, 시나리오 상단의 **ON/OFF 토글을 ON**으로 켜면 자동화가 시작됩니다. 이제 구글 시트에 새 행이 추가될 때마다 슬랙으로 자동 메시지가 발송됩니다.

기본 텍스트 메시지도 충분하지만, 슬랙의 Block Kit을 활용하면 섹션, 구분선, 버튼이 있는 카드형 메시지를 만들 수 있습니다.

### Block Kit이란?

Block Kit은 슬랙이 공식 제공하는 UI 컴포넌트 시스템입니다 (출처: [Slack 공식 Block Kit 문서](https://api.slack.com/block-kit)). JSON 형식으로 메시지를 구성하며, Make.com의 슬랙 모듈에서 'Blocks' 필드에 직접 입력할 수 있습니다.

### 실전 Block Kit 템플릿

아래 JSON을 Make.com의 슬랙 모듈 **Blocks** 필드에 붙여넣고, `{{1.컬럼명}}` 변수를 실제 필드에 맞게 교체하세요.

```json
[
  {
    "type": "header",
    "text": {
      "type": "plain_text",
      "text": "📊 일일 실적 자동 보고서",
      "emoji": true
    }
  },
  {
    "type": "section",
    "fields": [
      {
        "type": "mrkdwn",
        "text": "*날짜*\n{{1.날짜}}"
      },
      {
        "type": "mrkdwn",
        "text": "*담당자*\n{{1.담당자}}"
      },
      {
        "type": "mrkdwn",
        "text": "*항목*\n{{1.항목}}"
      },
      {
        "type": "mrkdwn",
        "text": "*매출*\n{{1.매출(원)}}원"
      }
    ]
  },
  {
    "type": "divider"
  },
  {
    "type": "context",
    "elements": [
      {
        "type": "mrkdwn",
        "text": "🤖 Make.com 자동 발송 | 데이터 원본: 구글 시트"
      }
    ]
  }
]
```

### 주간 요약 보고서로 업그레이드하기

더 고급스러운 활용법은 **Google Sheets → Aggregate → Slack** 3단계 워크플로우입니다.

1. **Google Sheets**: 한 주간의 데이터 행을 모두 읽어옴
2. **Aggregator 모듈**: 여러 행의 숫자를 합산·평균 계산
3. **Slack**: 집계된 주간 요약을 하나의 메시지로 전송

Aggregator는 Make.com의 내장 모듈로, 코딩 없이 SUM, COUNT, MAX 등을 계산할 수 있습니다. 이 구조면 "지난 주 총 매출: 8,400,000원 / 거래 건수: 7건 / 최고 담당자: 김민준" 같은 주간 요약 메시지를 완전 자동으로 슬랙에 발송할 수 있습니다.

> 💡 **실전 팁**: Aggregator 모듈 추가 후 반드시 'Group by' 필드를 설정하세요. 이를 생략하면 전체 데이터를 하나로 묶는 대신 행마다 개별 합산값을 계산해 이상한 결과가 나옵니다.

## 실제 활용 사례: 어떤 팀이 어떻게 쓰고 있나

노코드 자동화는 이미 다양한 규모의 팀에서 실무에 적용되고 있습니다.

디지털 마케팅 에이전시에서는 클라이언트별 구글 시트(광고 성과 데이터)를 Make.com으로 연동해, 매주 금요일 오후 클라이언트 슬랙 채널에 주간 성과 요약을 자동 발송하는 구조를 사용하는 것으로 알려져 있습니다. 클라이언트 수가 늘어도 Make.com 시나리오만 복제하면 되므로 확장성이 높다는 평가입니다.

### Make.com 공식 고객 사례

Make.com 공식 홈페이지에 따르면, 전 세계 다양한 기업이 HR 온보딩 자동화, 전자상거래 주문 알림, 고객 지원 티켓 라우팅 등에 Make.com을 활용하고 있습니다 (출처: [Make.com 공식 고객 사례](https://www.make.com/en/customers)). 특히 반복 수작업 제거를 통해 주당 수 시간의 업무 시간 절감 효과를 보고하는 사례가 다수 공개되어 있습니다.

## 초보자가 자주 빠지는 5가지 함정과 해결법

직접 Make.com을 사용하며 경험하거나, 커뮤니티에서 자주 목격되는 실수를 정리합니다.

### 함정 1: 구글 시트 연결 계정 혼선

가장 흔한 실수입니다. Make.com에 A 구글 계정을 연결했는데, 실제 시트는 B 계정 소유인 경우 권한 오류가 발생합니다. **Connections** 메뉴에서 올바른 계정으로 재연결하거나, 시트를 A 계정과 공유 설정하세요.

### 함정 2: 'From all' 설정으로 기존 데이터 폭탄

앞에서도 언급했지만, 'Watch Rows' 트리거에서 **'Where to start'를 'From all'로 설정하면** 기존에 입력된 수백~수천 행의 데이터가 한꺼번에 슬랙으로 발송됩니다. 처음에는 반드시 **'From now on'** 또는 **특정 행 번호**를 지정하세요.

### 함정 3: 시나리오 저장 후 ON 토글 미확인

Run once 테스트까지 성공했는데 자동화가 안 된다면, 시나리오 ON/OFF 토글이 꺼져 있을 가능성이 큽니다. **Save 후 반드시 토글을 ON으로 켜세요.** 이것이 가장 허탈한 실수 중 하나입니다.

### 함정 4: 슬랙 채널 ID vs 채널 이름 혼동

Make.com의 슬랙 모듈에서 채널을 선택할 때 드롭다운에서 선택하면 채널 ID가 자동으로 매핑됩니다. 그런데 채널 이름을 직접 텍스트로 입력(`#daily-report`)하면 인식 오류가 발생하는 경우가 있습니다. **항상 드롭다운에서 직접 선택하세요.**

### 함정 5: 무료 플랜 오퍼레이션 소진 후 자동화 중단

무료 플랜의 월 1,000 오퍼레이션을 다 쓰면 시나리오가 자동으로 멈춥니다. Make.com이 이메일로 알림을 보내지만, 놓치는 경우가 있습니다. **대시보드 Usage 탭을 주기적으로 확인하거나, 오퍼레이션 부족 알림 이메일 필터를 설정해두세요.** 오퍼레이션이 부족하면 $9의 Core 플랜 업그레이드를 검토하세요.

**Q1: Make.com 무료로 쓸 수 있나요? 유료 플랜이 필요한 경우는 언제인가요?**

A1: 네, Make.com은 무료 플랜(Free)을 제공합니다. 2026년 4월 기준 무료 플랜은 월 1,000 오퍼레이션(작업 실행 횟수)과 최대 2개의 활성 시나리오를 제공합니다. 이 글에서 만드는 구글 시트→슬랙 자동 보고서 수준의 자동화라면 무료 플랜으로도 충분히 운영 가능합니다. 단, 팀 전체가 공유하거나 하루에도 여러 번 트리거가 실행되는 환경이라면 월 오퍼레이션이 빠르게 소진될 수 있어요. 이 경우 월 $9의 Core 플랜으로 전환하면 월 10,000 오퍼레이션과 무제한 활성 시나리오를 사용할 수 있습니다. 처음에는 무료로 시작해 사용량을 확인한 뒤 업그레이드하는 전략을 추천합니다.

**Q2: n8n이랑 Make.com 차이가 뭔가요? 초보자한테는 어떤 게 더 좋나요?**

A2: 가장 큰 차이는 설치 여부와 진입 장벽입니다. n8n은 기본적으로 셀프 호스팅(서버에 직접 설치)이 필요하고, 클라우드 버전도 있지만 설정 과정이 Make.com보다 복잡합니다. 반면 Make.com은 브라우저에서 회원가입만 하면 즉시 시작할 수 있어 진입 장벽이 훨씬 낮습니다. 워크플로우 시각화 측면에서도 Make.com의 원형 모듈 방식이 직관적이라는 평가가 많습니다. 다만 n8n은 오픈소스라 서버만 있으면 무제한 무료로 쓸 수 있고, 고급 커스터마이징이 유리합니다. 코딩 경험이 없는 초보자에게는 Make.com이 더 적합합니다.

**Q3: Make.com 구글 시트 연동할 때 권한 오류가 자꾸 뜨는데 어떻게 해결하나요?**

A3: 구글 시트 연동 시 권한 오류는 크게 두 가지 원인에서 발생합니다. 첫 번째는 Make.com에 연결한 구글 계정과 실제 시트를 소유한 계정이 다를 때입니다. 이 경우 Connections 메뉴에서 구글 계정을 올바른 계정으로 재연결하세요. 두 번째는 구글 시트의 공유 설정이 적절하지 않을 때입니다. 시트 소유자 계정을 Make.com에 연결하거나, 해당 계정에 편집 권한을 부여하면 해결됩니다. 오류 코드 403이 뜬다면 구글 계정의 OAuth 접근 권한을 다시 부여해보세요.

**Q4: Make.com 워크플로우가 실행 안 될 때 어떻게 디버깅하나요?**

A4: Make.com에는 'Run once' 버튼과 실행 기록(Execution History)이 내장되어 있어 디버깅이 비교적 쉽습니다. 시나리오 편집 화면 하단의 'Run once'를 클릭하면 각 모듈이 순서대로 실행되고, 어느 단계에서 오류가 발생했는지 빨간 느낌표로 표시됩니다. 오류 모듈을 클릭하면 상세 에러 메시지와 전달된 데이터를 확인할 수 있습니다. 가장 흔한 원인은 연결(Connection) 만료, 슬랙 채널 ID 오입력, 구글 시트의 헤더 행 불일치입니다. 정기 실행 시나리오라면 Scheduling 탭에서 활성화(ON) 상태인지 반드시 확인하세요.

**Q5: Make.com 유료 플랜 가격이 얼마인가요? 팀으로 쓰면 더 비싸지나요?**

A5: 2026년 4월 공식 사이트 기준, Make.com의 주요 플랜 가격은 Free($0, 월 1,000 오퍼레이션), Core($9/월, 10,000 오퍼레이션), Pro($16/월, 고급 기능 포함), Teams($29/월, 팀 협업 기능)입니다. 연간 결제 시 약 20% 할인이 적용됩니다. 팀 플랜은 멤버 수가 아닌 오퍼레이션 사용량 기준으로 과금되므로, 팀원이 많아도 자동화 실행 횟수가 적다면 비용이 크게 늘지 않습니다. 정확한 최신 가격은 공식 사이트에서 확인하는 것을 권장합니다.
